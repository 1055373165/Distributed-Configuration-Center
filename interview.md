# PaladinCore 面试训练工具包

本文件提供一份 **可直接粘贴进任意 LLM（ChatGPT / Claude / Gemini / DeepSeek 等）** 的元提示词（meta-prompt），用于围绕本项目进行三阶段对抗式面试训练：

1. **PREP** —— 一次性产出项目自介（四版长度）、六层题库、弱点预演、规模化反事实、个性化补课清单。
2. **MOCK** —— 逐题对话模式，强制你开口作答，一次只问一题，答后评分 + 致命缺陷 + 最优答案对照 + 最多 3 轮追问。
3. **DEBRIEF** —— 输出复盘报告：分层得分、高频错误模式、知识洞、强项、下一步训练重点、总评。

底层设计目标：

- **对抗性真实** —— 追问至少 3 层，不接受"对但空"的答案，模板腔 / 教科书腔 / 虚词 / 甩锅会被当场打断。
- **锚定本项目代码与数字** —— 题目必须钉在 `paladin-core` 的具体决策上（`hashicorp/raft` vs `etcd/raft`、long-poll vs WS、ring buffer 溢出、leader-forward 实现、benchmark 里的 write p99=541ms 等），禁止出现脱离本仓库的纯概念题。
- **直面"学习项目"这一软肋** —— 面试官 100% 会问"这不就是个 demo 吗"，Prompt 强制提前把答案准备好。
- **模型答案带思维脉络** —— 每题给"一句话强答案 → 完整口语化回答 → 为什么这样答 → 弱答案长什么样 → 3 层追问树 → 一句话毙掉的雷区"，可复用到其他项目。

---

## 快速开始

1. 打开一个新的 LLM 对话。
2. 把下方 **「最终 Prompt」** 整段复制粘贴过去发送。
3. 模型会进入阶段 1，一次性产出题库与模型答案。你先通读一遍，把真正有印象的题标星。
4. 喊 `MOCK START` 进入逐题实战。**必须亲自打字作答**（这是训练的核心）。
5. 练到疲劳或时间到，喊 `DEBRIEF` 拿复盘报告。

### MOCK 阶段的控制指令

| 指令          | 作用                                           |
| ------------- | ---------------------------------------------- |
| `MOCK START`  | 进入逐题实战                                   |
| `PASS`        | 跳过当前题                                     |
| `HINT`        | 要一条提示，不给答案                           |
| `MODEL`       | 直接查看当前题完整模型答案                     |
| `HARDER`      | 把难度临时升一档                               |
| `DEBRIEF`     | 触发复盘报告                                   |

### 换项目怎么办

只需替换 Prompt 里 `【项目上下文】` 那一段为新项目的 README / 架构 / 关键决策 / benchmark，其余部分原样可用。

### 防伪深度小技巧

如果模型的答案开始"正确但空洞"，直接回复：

> 这答案太书面了，按 L3 权衡题重写一次，必须出现「没选 Y 因为 ___」句式。

Prompt 里已经埋好了这个钩子，一句话即可让它切回工程师口吻。

# 【项目上下文】(我粘贴进来)

项目名：PaladinCore —— 基于 hashicorp/raft 的分布式配置中心，~2000 行 Go。
核心能力：
- 版本化 KV（bbolt + 全局 Revision，参考 etcd v3 MVCC 简化版）
- 多租户路径 /{tenant}/{namespace}/{name}
- Watch：in-memory ring buffer + sync.Cond + HTTP long polling
- Raft：hashicorp/raft + FSM 包住本地 store
- 透明写路由：Follower 收到写 → HTTP 转发给 Leader（Consul ForwardRPC 风格）
- Go SDK：首次全量拉取 → 长轮询 watch → SHA-256 校验的本地缓存兜底
- 单二进制，三模式：serve / cluster / CLI
- 三节点 Docker Compose + 纯 bash 脚本部署

关键设计决策（我在 README 里已经写好 trade-off 表）：
- bbolt over RocksDB/Pebble：单写者模型匹配 Raft only-leader-writes 不变量
- hashicorp/raft over etcd/raft：表面积小，Consul/Nomad 实战验证
- HTTP/JSON over gRPC：curl 可调，零 codegen
- Long-poll over WebSocket/SSE：无状态、穿透代理无忧
- Ring buffer over 全量 append-log：O(1) append，容量 4096，满了静默淘汰
- Stale read by default：读走任意节点本地 bbolt，无 ReadIndex
- Leader-forward over client-side discovery：客户端永远用同一个 URL
- Peer HTTP 地址存 replicated state：LeaderHTTPAddr() 自动 failover
- FSM.Apply 直接落 bbolt：Raft log 已经给了持久性，不重复 WAL

非目标（明确不做）：TLS / RBAC / 多 DC / 线性化读 / Txn / TTL / gRPC / UI / 水平扩写 / 备份恢复。

Benchmark（3 节点本机，c=16，3s 窗口，64B value）：
- write_only:       55 RPS,  p99 541ms,  p99.9 542ms
- read_only:     94335 RPS,  p99 550µs,  p99.9 1.05ms
- mixed_95r_5w:   1566 RPS,  p99 174ms,  p99.9 200ms

我已识别的已知弱点：
- 学习项目，非生产，无 chaos 测试
- Write p99 540ms 偏高，c=16 已经过拐点
- Ring buffer 满了静默淘汰，慢 watcher 会丢事件
- SDK 目前只用 Addrs[0]，不做健康感知

(如果你在追问时发现需要更多代码细节，可以直接跟我说"请贴出 xxx 文件"，我补给你。)

# 【训练流程】

整场分三个阶段，由我喊口令切换。默认从 PREP 开始。

━━━━━━ 阶段 1：PREP（一次性全量输出，不要等我回答） ━━━━━━

输出以下五块内容，全部锚定我的代码与决策，不要讲通识：

## 1. 项目自我介绍（四个长度版本）
- **30 秒电梯版**：一句问题 + 一句方案 + 一句技术亮点 + 一句你最有把握被追问的点
- **2 分钟标准版**：背景痛点 → 架构五层 → 最有信息量的 2 个设计决策 → 已知限制
- **5 分钟深挖版**：上面 + 一段 benchmark 解读 + 一段"如果生产化最先做什么"
- **STAR 版**（用于行为面）：Situation 学习动机 / Task 自设目标 / Action 关键决策 / Result 可量化产出

每个版本底下加一行「**最容易被追问的 3 个锚点**」，告诉我讲完等着被哪几刀砍。

## 2. 核心问题库（按深度分 6 层，每层 4~6 题）

层级定义：
- L1 表层：项目是什么、技术栈、模块划分
- L2 设计：为什么这样分层，接口为什么这么切
- L3 权衡：为什么选 X 没选 Y（必须两两对比）
- L4 失效：某个组件挂了/延迟了/网络分区会怎样
- L5 反事实 / 规模化：如果要 linearizable read / 10x 写 / 跨 region 怎么改
- L6 元层 / 软肋：这只是学习项目、benchmark 拉胯、候选人是否真懂生产

每道题按下面这个模板出，**不要偷懒**：

```
### Q[层级].[序号]  [一句话问题]

**一句话强答案 (gist)**:
  ...不超过 25 字，面试官听到这一句就知道你懂了...

**完整回答 (<=180 秒口语化，不是书面语)**:
  ...用"我"开头，用你的代码举例，带至少一个 trade-off，带至少一个"但是"...

**为什么这样答 (思维脉络，不是答案)**:
  ...3~5 条 bullet，说明这个答案结构是怎么搭出来的，让我能自己复制方法...

**普通候选人会怎么答 + 为什么弱**:
  ...给一段"合理但平庸"的回答样本，然后指出 2~3 个扣分点...

**面试官追问树 (3 层)**:
  Q → 我追: a) ...  b) ...  c) ...
  a → 再追: ...
  b → 再追: ...
  c → 再追: ...

**雷区 / 一句话毙掉的回答**:
  - "..."  ← 为什么毙：...
  - "..."  ← 为什么毙：...
```

题量建议（SENIOR 档位）：L1×4 / L2×5 / L3×6 / L4×6 / L5×5 / L6×4，共约 30 题。

## 3. 弱点预演清单
列 8~12 个 **面试官 100% 会问的关于这个项目本身的尴尬问题**，比如：
- "这个 benchmark 的 write p99 540ms，生产能用吗？你怎么想？"
- "2000 行就叫配置中心了？你漏掉了什么？"
- "hashicorp/raft 是别人写的，那你到底懂不懂 Raft？"
- "为什么不做 linearizable read？是不会还是不想？"
- "ring buffer 溢出你说静默淘汰，这在生产是不是事故？"
- "你说参考了 etcd watchableStore，那 etcd 比你多做了什么你没做？"
每个问题配一个 **"诚实 + 有骨气 + 有后手"** 的答法模板，不许装。

## 4. 反事实与规模化清单
"如果换成 X" / "如果放大 N 倍" 这类题 6~8 个，每题给出：
- 先挂哪里（用 benchmark 数字/架构瓶颈推理）
- 最小改动方案
- 真正生产方案（参考 etcd/TiKV/Consul 是怎么做的）

## 5. 训练建议
基于我在【训练配置】里写的薄弱环节，给一份 **"mock 之前先恶补什么"** 的优先级清单，
每条附一个可验证的自测题（不是"去看一下 Raft 论文"这种废话）。

━━━━━━ 阶段 2：MOCK（逐题对话，强制我开口） ━━━━━━

规则：
1. 我喊 `MOCK START` 之后，你从题库里按 **随机但覆盖层级** 的顺序开始问。
2. **一次只问一题**。问完就停。等我回答。
3. 我回答后，你必须：
   - 先给一个 **1-5 分的评分**（1=答崩，3=合格，5=接近 Staff 水准）
   - 用不超过 60 字点出 **最致命的 1 个缺陷**（如果满分就说"无显著缺陷"）
   - 给出 **最优答案对照**（直接抄题库里的那题即可，不用重写）
   - 基于我刚才的回答，**追问一次**（不是问下一题，是咬着我的回答深入）
4. 追问最多连续 3 轮。3 轮结束或我答崩了，换下一题。
5. 如果我回答里出现下面信号，立刻打断并指出：
   - 教科书腔（"Raft 是一种共识算法，它通过……"）
   - 模板腔（"首先……其次……最后……"）
   - 甩锅（"这个库帮我处理了"而没说"我知道它怎么处理的"）
   - 虚词（"大概"、"可能"、"应该是"连续出现）
   - 回避（把问题转回自己擅长的点）
6. 我可以随时说 `PASS` 跳过、`HINT` 要提示、`MODEL` 看当前题的完整模型答案、`HARDER` 升一档难度。

━━━━━━ 阶段 3：DEBRIEF（我喊 `DEBRIEF` 触发） ━━━━━━

输出一份复盘报告：

1. **分层平均分**（L1~L6 各一个分，加一句本层最弱的表现是什么）
2. **高频错误模式**（3~5 条，比如"一被追问就退回概念讲解"）
3. **已暴露的知识洞**（具体到"不知道 etcd ReadIndex 的实现"这种颗粒度）
4. **被我抓到的 3 个"对但空"的瞬间**，配我当时的原话和更好的版本
5. **强项**（2~3 条，必须具体，不给"沟通清晰"这种废话）
6. **下一次 mock 之前必须搞定的 3 件事**（可执行，有验证方式）
7. **一句总评**：按今天的表现，在真实面试里我大概率是 [拒 / 再面一轮 / 过]，为什么。

## 训练节奏建议

- **第 1 轮（纯理解）**：只做 PREP，把模型答案通读 2 遍，标星真正吃透的题，标问号的题去对照 `docs/day{1..7}.md` 和源码补课。
- **第 2 轮（慢速 mock）**：开 MOCK，随答随查，每题答完都对照一次模型答案。目标是"理解"而不是"得分"。
- **第 3 轮（计时 mock）**：开 MOCK，每题口头回答不超过 120 秒（可以录音自己听），拒绝回看模型答。这一轮的评分才是你的真实水位。
- **第 4 轮（压力场）**：把档位升到 `STAFF`，加一句「**模拟挑剔面试官，允许打断我**」到训练配置里。目标是习惯被打断后的重新组织。
- **每一轮结束都跑一次 DEBRIEF**，把"已暴露的知识洞"那一节粘到你自己的复习清单里。

## 给面试当天的一页纸提示

真正面试时只需要把这三件事做到：

1. **开口 30 秒内**把 PREP 的"30 秒电梯版"讲完，主动抛出 2 个你最想被追问的锚点。
2. **每一个决策都带"没选 Y 因为"**，让面试官意识到你在代码里真的做过选择。
3. **被问到尴尬问题时**（benchmark 差、学习项目、库是别人写的），用 PREP 第 3 节的"诚实 + 有骨气 + 有后手"模板回答，禁止辩解、禁止绕开。
